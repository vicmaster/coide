# Contributing

Issues and pull requests are welcome.

- For larger changes, open an issue first to discuss the approach
- Keep PRs focused — one feature or fix per PR
- Set up locally with `npm install` then `npm run dev`
- Run `npm test` before submitting
- Match the existing code style — TypeScript strict, Tailwind v3, Zustand for stores
- Conventional-style commit messages preferred (`Add X`, `Fix Y`, `Update Z`)
